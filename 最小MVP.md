# MVP 需求定义

这是一个**本地桌面操作录制与回放**的最小可用产品。

用户在本地桌面完成一次真实操作后，系统能够准确记录鼠标、键盘、时间顺序和屏幕位置，并将这段操作沉淀为可复用的 flow / skill。后续在相同环境下，系统可以按原始动作进行高还原回放，实现对用户操作的直接复刻。

## 核心目标

### 1. 记录真实操作，而不是理解内容

MVP 不做视觉识别、不做界面理解、不做条件判断，重点只放在**记录和复现用户动作本身**。

### 2. 强调高还原、强一致

系统本质上是：

- 记住位置
- 记住点击
- 记住输入
- 记住时间顺序

目标不是"智能执行"，而是尽可能原样复刻。

### 3. 严格依赖环境一致性

该 MVP 默认运行在固定分辨率、固定窗口位置、固定应用界面下。
一旦位置变化、窗口变化、分辨率变化，系统不做自适应修正，直接提示用户重新录制。

## MVP 核心能力

### 操作录制

支持录制本地桌面的基础输入事件，包括：

- 鼠标移动
- 鼠标点击 / 按下 / 抬起
- 鼠标滚轮
- 键盘按键
- 文本输入
- 操作间隔时间

### Flow / Skill 生成

将录制结果自动转换为结构化 flow，并保存为可重复调用的 skill。

### 操作回放

按照录制时的时间顺序、坐标位置和输入动作，完整回放用户操作过程。

### 环境校验

在执行 skill 前，校验运行环境是否与录制环境一致，包括：

- 系统平台
- 屏幕分辨率
- DPI / 缩放
- 窗口标题
- 窗口位置与大小

若不一致，直接终止执行并提示重新录制。

## 重点边界

### 本期明确不做

- 不做图像识别
- 不做元素识别
- 不做 OCR
- 不做页面状态判断
- 不做异常分支处理
- 不做位置漂移纠偏
- 不做跨环境适配
- 不做智能决策

### 本期只做

- 录制动作
- 存储 flow / skill
- 在同环境下高精度回放

## 产品本质

这个 MVP 的本质不是 AI 助手，也不是智能 Agent。
它首先是一个**本地桌面动作录制器 + Flow 播放器 + Skill 存储器**。

它解决的核心问题是：

> 把用户一次真实操作，沉淀成一个可重复执行、可复刻的本地桌面 skill。
